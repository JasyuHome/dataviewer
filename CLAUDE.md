## Project Overview

Full-stack web application for CSV data management, visualization, and Notion integration.

**Tech Stack:**
- **Frontend:** React 18 + TypeScript + Ant Design + ECharts, built with Vite
- **Backend:** Go 1.21+ with Gin framework, SQLite database
- **Integrations:** Notion API (using Resty HTTP client)

## Commands

### Backend

```bash
cd backend
go mod tidy
go run main.go              # Start server on port 8080 (or $SERVER_PORT)
```

### Frontend

```bash
cd frontend
pnpm install
pnpm dev                    # Start dev server on port 3000
pnpm build                  # Production build
pnpm lint                   # Run ESLint
```

## Architecture

### Backend (`backend/`)

```
backend/
├── main.go              # Entry point, Gin router setup, CORS, API routes
├── config/
│   └── config.go        # Environment configuration
├── handlers/
│   ├── upload.go        # CSV file upload, list, delete, rename
│   ├── query.go         # Data query with filters, pagination, sorting
│   ├── charts.go        # Chart generation (bar, line, pie)
│   └── notion.go        # Notion integration (list, query, create pages)
├── services/
│   ├── database.go      # SQLite operations, dynamic table creation
│   ├── csv.go           # CSV parsing, delimiter detection, type inference
│   └── notion.go        # Notion API client (search, databases, pages)
└── models/
    └── metadata.go      # Data structures (FileMetadata, ColumnDef, QueryResult)
```

**Key Backend Features:**
- Auto-detects CSV delimiters (comma, semicolon, tab, Chinese comma)
- Infers column types (INTEGER, REAL, TEXT, DATETIME)
- Creates dynamic SQLite tables from CSV files
- Supports currency symbols and formatted numbers in CSV data

### Frontend (`frontend/`)

```
frontend/src/
├── App.tsx              # Main layout with navigation menu
├── main.tsx             # Entry point
├── types/
│   └── index.ts         # TypeScript interfaces
├── services/
│   └── api.ts           # Axios-based API client
├── pages/
│   ├── Home.tsx         # Landing page with file upload
│   ├── DataManage.tsx   # Table list, rename, delete, export
│   ├── DataQuery.tsx    # Visual query builder with conditions
│   ├── DataViz.tsx      # Chart creation (bar, line, pie)
│   ├── Notion.tsx       # Notion integration (browse, query, create)
│   └── Settings.tsx     # Configuration page
└── components/
    └── FileUpload/      # Drag-and-drop upload component
```

**Pages:**
- `/` - Home with file upload
- `/manage` - Data management (list, rename, delete, export)
- `/query` - Query builder with AND/OR logic, operators (=, ≠, >, <, ≥, ≤, contains, range)
- `/viz` - Data visualization with ECharts
- `/settings` - Application settings

## API Endpoints

### Core API

| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/api/upload` | Upload CSV file |
| GET | `/api/files` | Get file list |
| DELETE | `/api/files/:id` | Delete file |
| PUT | `/api/files/:id/rename` | Rename file |
| POST | `/api/query` | Execute query with conditions |
| GET | `/api/tables/:tableName/data` | Get table data |
| GET | `/api/tables/:tableName/structure` | Get table structure |
| GET | `/api/tables/:tableName/export` | Export as CSV |
| POST | `/api/charts/generate` | Generate chart |
| GET | `/api/charts/:tableName/data` | Get chart data |

### Notion API

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/notion/databases` | List accessible databases |
| GET | `/api/notion/databases/:id` | Get database details |
| POST | `/api/notion/databases/:id/query` | Query database with pagination |
| POST | `/api/notion/pages` | Create new page |
| PUT | `/api/notion/pages/:id` | Update page properties |
| DELETE | `/api/notion/pages/:id` | Archive (soft delete) page |
| GET | `/api/notion/search` | Search pages |

## Data Flow

1. CSV upload → parsed by `CSVService` → columns analyzed for types
2. `DatabaseService` creates dynamic table with inferred schema
3. Frontend queries data via `api.ts` → backend executes SQL → returns paginated results
4. Charts generated by aggregating table data based on x/y field mapping

## Configuration

Environment variables (backend):
- `SERVER_PORT` (default: 8080)
- `DATABASE_PATH` (default: ./storage/dataviewer.db)
- `STORAGE_PATH` (default: ./storage/uploads)
- `NOTION_API_KEY` - Notion integration API key (required for Notion features)
- `NOTION_VERSION` (default: 2022-06-28) - Notion API version
